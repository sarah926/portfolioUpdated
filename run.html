Run Buddy


A mobile app to help users kickstart and maintain their running journey.



Product Analysis:

See on Github

Customer Problem: Users find it difficult to sustain their interest in running due to hidden paywalls and non-intuitive interfaces.

KPI: Sign up a minimum of 50 users, with 50% of them completing more than 5 tracked workouts.

Goals:

  • Develop an iOS app with options for Apple Watch and IPhone.
  • Provide a user friendly interface to easily track workouts and follow a beginner program.
  • Create a better user experience with no paywalls compared to other apps in the market.
Role: Software developer, product analyst. Technologies Utilized: SwiftUI, Figma. User Journey:
  1. User downloads the application from the apple app store.
  2. The user is given one or 2 screens of introduction of the app and its purpose.
  3. The user enters personal information to set up an account.
  4. The user selects what level of runner they are - beginner, intermediate, or advanced.
  5. The user is taken to a home page, where they will be able to see their data after they complete their first workout.
  6. The user can easily click start workout when they are ready.
  7. The user can see the interval information on their phone, including the time left and if it is a walk or run interval.
  8. If the user turns off their phone while they are running, the app will beep so they know it is time to switch.
  9. After the workout is done, the user will be able to see a summary.
